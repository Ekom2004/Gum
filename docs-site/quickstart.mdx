## 1. Install the SDK

```bash
pip install usegum
```

## 2. Initialize a Gum project

Run `gum init` from the project root. It creates `gum.toml`, `.env.example`, a minimal `pyproject.toml` if one does not exist, and a starter `jobs.py`.

```bash
gum init --project-id proj_live --api-base-url https://api.your-gum-cloud.com
```

`gum.toml` stores non-secret project defaults.

```toml
project_id = "proj_live"
api_base_url = "https://api.your-gum-cloud.com"
```

Set your API key separately.

```bash
export GUM_API_KEY="gum_live_..."
```

For local development, Gum defaults to `http://127.0.0.1:8000`.

## 3. Write a function

Create `jobs.py`.

```python
import gum

openai_limit = gum.rate_limit("60/m")

@gum.job(retries=5, timeout="5m", cpu=2, memory="1gb", rate_limit=openai_limit, concurrency=5, key="customer_id")
def sync_customer(customer_id: str):
    crm.sync(customer_id)
```

## 4. Deploy to Gum

Run deploy from the project root.

```bash
gum deploy
```

Gum discovers decorated functions, packages your code, and registers the deploy.

## 5. Enqueue work

Call `.enqueue(...)` from application code.

```python
run = sync_customer.enqueue(customer_id="cus_123")
print(run.id, run.status)

# delay execution by 10 minutes
sync_customer.enqueue(customer_id="cus_123", delay="10m")
```

If another enqueue uses the same `customer_id` while the key is active, Gum returns the existing run instead of creating duplicate work.

## 6. Inspect the run

```bash
gum get run_123
gum logs run_123
gum replay run_123
```

For operators:

```bash
gum admin
```
