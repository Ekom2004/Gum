---

## `GUM_API_BASE_URL`

Base URL for the Gum API.

```bash
export GUM_API_BASE_URL="https://api.your-gum-cloud.com"
```

Default:

```bash
http://127.0.0.1:8000
```

You can also store this non-secret value in `gum.toml`.

```toml
api_base_url = "https://api.your-gum-cloud.com"
```

## `GUM_PROJECT_ID`

Project ID used by `gum deploy`.

```bash
export GUM_PROJECT_ID="proj_live"
```

You can also store this non-secret value in `gum.toml`.

```toml
project_id = "proj_live"
```

## `GUM_API_KEY`

API key used for normal SDK and CLI requests.

```bash
export GUM_API_KEY="gum_live_..."
```

## `GUM_ADMIN_KEY`

Admin key used for operator-only views.

```bash
export GUM_ADMIN_KEY="gum_admin_..."
```

You can also store the admin key locally:

```bash
gum admin login
```

## `GUM_RUNNER_MEMORY_MB`

Memory capacity reported by a Gum runner.

```bash
export GUM_RUNNER_MEMORY_MB="4096"
```

Jobs with `memory="4gb"` can only lease onto runners with enough remaining memory capacity.

## `GUM_RUNNER_MAX_CONCURRENT_LEASES`

Maximum active leases a runner should take at once.

```bash
export GUM_RUNNER_MAX_CONCURRENT_LEASES="4"
```
---
title: "Statuses"
description: "Run statuses shown by the SDK, CLI, and admin console."
---

Runs move through a small public status set.

## `queued`

The run exists but is not executing yet.

Common reasons:

- waiting for concurrency
- waiting for rate limit
- waiting for retry delay
- waiting for function health recovery

## `running`

An attempt is actively executing.

## `succeeded`

The run completed successfully.

## `failed`

The run failed and has no retry budget left, or the failure is terminal.

## `timed_out`

The attempt exceeded its timeout. The run may retry if retry budget remains.

## `canceled`

The run was canceled by an operator or API request.

## Waiting reasons

Gum may include waiting or failure details such as:

- `waiting_on_concurrency`
- `waiting_on_rate_limit`
- `waiting_for_function_health_recovery`
- `downstream_unavailable`
